## 概要
* Rails を中心とした Web バックエンド開発者（フリーランス）です
* 業務ロジックの実装と、RSpec を用いたテストコード実装が好きで、これらを得意分野としています
* 要件がふわっとしていても、詳細設計（＆レビュー） 〜 実装（＆テスト）をバイネームで遂行することができます
* 自分の手を止めてでもコードレビューする派です
  - Pull Request の滞留時間が、チーム全体の生産性や効率、及びレビューイーのモチベーションに大きな影響を及ぼすと考えているからです
  - メンバーがどんなコードを書くのか？”自分だったらこう書く”との違いは何か？の好奇心が先行するからです
  - コードレビューしない人は、解り易い Pull Request が書けないと確信しています（ガイドラインが整備されているとしても）
* 全てのアウトプットに関して「数年後に読んで理解できるか」「当事者以外が読んで理解できるか」を常に意識しており、属人性の排除にも貢献してきました
* 最近は副業を通じて、モダンな Rails と AI エージェント周りの技術に触れています
