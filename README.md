## 概要
* Ruby on Rails による Web バックエンド開発者（フリーランス）です
* 業務ロジックの実装と、RSpec を用いたテストコード実装が好きで、これらを得意分野としています
* 要件がふわっとしていても、詳細設計（＆レビュー） 〜 実装（＆テスト）をバイネームで遂行することができます
* 自分の手を止めてでも最優先でコードレビューする派です
  - Pull Request の滞留時間がチーム全体の生産性と、レビューイーのモチベーションに大きな影響を及ぼすと考えているからです
  - 「メンバーがどんなコードを書くのか？」「”自分だったらこう書く”との違いは何か？」の好奇心が根底にあるからです
  - コードレビューしない人は、スキルアップの機会を逃していると共に、解り易い Pull Request が書けないと確信しています（故にチームのレビュー文化を重要視します）
  - 自分の進捗が遅れると元も子もないので、そこはいー感じに振る舞います
* 全てのアウトプットに関して「数年後に読んで理解できるか」「当事者以外が読んで理解できるか」を常に意識しており、属人性の排除にも貢献してきました（大型連休明けの自分に対しても）
* 書いたコードで悦に浸れる性格ですが、クライアントが掲げるゴールに寄与できなければ只の自己満足になってしまうと考え、臨機応変に妥協案を選択することも厭いません
* 直近では副業を通じて、モダンな Rails と AI エージェント周りの技術に触れていま~す~した（本業にフルパワーを注いでいます）
